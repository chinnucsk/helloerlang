这里没用对 进程字典 的访问速度进行比较, 是想避免半进程消息堆积过多造成的瓶颈

* mochiglobal的适用场合,存储的只读数据较大时,速度比ets快,
  原理是,ets需要进行进程间数据的copy,数据量越大,效率越低
